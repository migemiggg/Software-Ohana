# MINUTA DE REUNIÓN SCRUM

## Inicio del Proyecto Bebe

**Proyecto:** Proyecto Bebe - Sistema de Gestión de Inventario, Producción y Distribución Ohana
**Fecha:** 25 de mayo de 2026
**Scrum Master:** Adal Gastelum Salazar
**Equipo de Desarrollo:** Paul, Efren y Jair
**Tipo de reunión:** Kick-off / Inicio de proyecto

# 1. Objetivo de la reunión

Definir el alcance inicial del proyecto Bebe, acordar roles del equipo, establecer entregables del primer sprint y documentar las primeras tareas de desarrollo.

# 2. Alcance inicial definido

### Módulos principales

* Gestión de clientes.
* Gestión de productos.
* Gestión de inventario.
* Gestión de recetas y producción.
* Gestión de ubicaciones y mapa interactivo.
* Gestión de pedidos.
* Control de usuarios y accesos.

### Prioridades del proyecto Bebe

* Definir la lógica de inventario para insumos y productos terminados.
* Normalizar los datos de clientes y ubicaciones.
* Generar un flujo inicial de pedidos vinculado al mapa.
* Implementar una interfaz de búsqueda de productos.
* Preparar el esquema inicial de la base de datos.

# 3. Decisiones tomadas

* El proyecto se llamará internamente "Proyecto Bebe" y quedará registrado como parte del sistema Ohana.
* Se trabajará en base a un sprint inicial con énfasis en inventario y mapa de ubicaciones.
* Se acordó que el primer entregable tendrá un prototipo funcional básico para pruebas internas.
* Se asignó a Adal como Scrum Master y se definieron listas de tareas por módulo.

# 4. Primeras tareas del sprint inicial

1. Configurar el repositorio y el entorno de desarrollo.
2. Crear el esquema inicial de la base de datos en `src/db/schema.sql`.
3. Definir las tablas para clientes, ubicaciones, productos e inventario.
4. Diseñar el flujo de creación y edición de productos.
5. Iniciar desarrollo del módulo de ubicaciones y su integración con el mapa.
6. Crear usuarios de prueba y validar permisos básicos.

# 5. Próximos pasos

* Iniciar la implementación del CRUD de productos y clientes.
* Avanzar en la normalización de clientes y ubicaciones.
* Definir el primer prototipo del mapa interactivo.
* Planear la siguiente reunión de seguimiento para revisar avances del sprint.
